# gstack agents

Библиотека навыков (skills) для AI-агентов в экосистеме **gstack**. Каждый навык — это структурированная инструкция, которую агент читает и выполняет при вызове соответствующей slash-команды.

## Структура проекта

```
.agents/skills/       — навыки: по папке на каждый, точка входа — SKILL.md
.cursor/rules/        — правила Cursor для интеграции slash-команд
```

## Навыки

### Планирование и стратегия

| Команда | Назначение |
|---------|------------|
| `/office-hours` | Брейншторм и валидация идей в стиле YC Office Hours (стартап или builder-режим) |
| `/plan-ceo-review` | Стратегическое ревью плана: расширение/сужение scope, поиск 10-звёздочного продукта |
| `/plan-eng-review` | Инженерное ревью: архитектура, edge cases, тесты, производительность |
| `/plan-design-review` | Дизайн-ревью плана: оценки 0–10 по каждому измерению, правки до «десятки» |
| `/autoplan` | Автоматический конвейер всех трёх ревью (CEO → design → eng) с авто-решениями |

### Дизайн

| Команда | Назначение |
|---------|------------|
| `/design-consultation` | Создание дизайн-системы с нуля: эстетика, типографика, цвет, layout. Генерирует `DESIGN.md` |
| `/design-review` | Визуальный QA живого сайта: находит проблемы со spacing, иерархией, AI-слопом и исправляет их |

### Разработка и отладка

| Команда | Назначение |
|---------|------------|
| `/investigate` | Систематическая отладка: investigate → analyze → hypothesize → implement. Без фикса без root cause |
| `/codex` | Обёртка Codex CLI: независимое ревью, adversarial challenge, консультация |

### Тестирование и QA

| Команда | Назначение |
|---------|------------|
| `/qa` | QA-тестирование веб-приложения с исправлением найденных багов (Quick / Standard / Exhaustive) |
| `/qa-only` | Только отчёт по QA без изменений кода |
| `/browse` | Headless-браузер для навигации, скриншотов, взаимодействия с элементами (~100 мс на команду) |
| `/benchmark` | Детекция регрессий производительности: baseline, Core Web Vitals, размеры ресурсов |

### Безопасность

| Команда | Назначение |
|---------|------------|
| `/cso` | Аудит безопасности: OWASP Top 10, STRIDE, секреты, CVE, supply chain |
| `/careful` | Предупреждения перед деструктивными командами (`rm -rf`, `DROP TABLE`, `force-push` и т.д.) |
| `/freeze` | Ограничение правок одной директорией на время сессии |
| `/unfreeze` | Снятие ограничения `/freeze` |
| `/guard` | Комбинация `/careful` + `/freeze` для максимальной безопасности |

### Code review и шипинг

| Команда | Назначение |
|---------|------------|
| `/review` | Pre-merge ревью PR: SQL-безопасность, границы доверия к LLM, побочные эффекты |
| `/ship` | Тесты → VERSION bump → CHANGELOG → коммит → PR |
| `/land-and-deploy` | Merge PR → CI → деплой → проверка продакшена через canary |
| `/canary` | Пост-деплой мониторинг: ошибки консоли, регрессии, скриншоты vs baseline |
| `/setup-deploy` | Конфигурация деплоя (Fly.io, Vercel, Render и др.) в `CLAUDE.md` |

### Документация и ретро

| Команда | Назначение |
|---------|------------|
| `/document-release` | Обновление README, ARCHITECTURE, CONTRIBUTING, CHANGELOG после релиза |
| `/retro` | Недельный инженерный ретро по коммитам, паттернам и метрикам кода |

### Утилиты

| Команда | Назначение |
|---------|------------|
| `/setup-browser-cookies` | Импорт cookies из реального браузера в headless-сессию для тестирования аутентифицированных страниц |

## Как это работает

1. Правило Cursor (`.cursor/rules/gstack.mdc`) регистрирует все slash-команды.
2. Когда пользователь вводит команду (например, `/ship`), агент читает соответствующий `SKILL.md` из `.agents/skills/ship/`.
3. `SKILL.md` содержит пошаговую инструкцию, которую агент выполняет автономно.

## Технологии

- **Markdown** — все навыки и чеклисты
- **TypeScript** — реализация навыка `browse` (headless-браузер)
- **Shell** — вспомогательные скрипты (`careful`, `freeze`, сборка `browse`)
- **Bun** — среда для генерации документации навыков (`bun run gen:skill-docs`)
